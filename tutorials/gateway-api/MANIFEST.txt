🔧 EXAMPLE MANIFESTS (Apply these to learn)
═══════════════════════════════════════════════════════════════════════════════

1. 06-sample-apps.yaml (2 KB) - TEST APPLICATIONS
   Purpose: Deploy sample applications for testing routing
   Contents:
   ├─ http-app Deployment (2 replicas)
   ├─ http-app Service
   ├─ api-app Deployment (1 replica)
   └─ api-app Service
   
   Use: kubectl apply -f 06-sample-apps.yaml
   When: Start with this, needed for all other examples
   Testing: kubectl get pods -n demo-apps

2. 01-basic-gateway.yaml (1 KB) - BASIC HTTP ROUTING
   Purpose: Learn the basics of Gateway and HTTPRoute
   Contents:
   ├─ Gateway "demo-gateway" (port 80, HTTP)
   └─ HTTPRoute "http-app-route" (routes demo.localhost to http-app)
   
   Use: kubectl apply -f 01-basic-gateway.yaml -n demo-apps
   When: After deploying sample apps
   Testing: curl -H "Host: demo.localhost" http://localhost:8080/

3. 02-advanced-routing.yaml (3 KB) - ADVANCED ROUTING PATTERNS
   Purpose: Learn path, header, and hostname-based routing
   Contents:
   ├─ Path-based routing (different routes for /api vs /)
   ├─ Header-based routing (X-Version header matching)
   └─ Multiple hostname routing (app.localhost vs api.localhost)
   
   Use: kubectl apply -f 02-advanced-routing.yaml -n demo-apps
   When: After basic gateway is working
   Testing: Multiple curl examples with different paths and headers

4. 03-tls-gateway.yaml (3 KB) - TLS/HTTPS TERMINATION
   Purpose: Learn HTTPS and certificate management
   Contents:
   ├─ Updated Gateway with HTTPS listener
   ├─ TLS certificate reference
   ├─ HTTPS HTTPRoute
   └─ Optional HTTP redirect route
   
   Prerequisites: TLS secret (demo-tls) must exist
   Setup:
   - openssl req -x509 -newkey rsa:4096 -keyout key.pem -out cert.pem ...
   - kubectl create secret tls demo-tls --cert=cert.pem --key=key.pem
   
   Use: kubectl apply -f 03-tls-gateway.yaml -n demo-apps
   When: After understanding basic routing
   Testing: curl -k https://localhost:8443/ (with -k for self-signed)

5. 04-cross-namespace.yaml (3 KB) - CROSS-NAMESPACE ROUTING
   Purpose: Learn routing to services in different namespaces
   Contents:
   ├─ api-backend namespace creation
   ├─ backend-app Deployment in api-backend
   ├─ backend-app Service in api-backend
   ├─ ReferencePolicy for security (in api-backend namespace)
   └─ HTTPRoute from demo-apps referencing backend-app service
   
   Use: kubectl apply -f 04-cross-namespace.yaml
   When: After understanding basic routing
   Testing: curl -H "Host: backend.localhost" http://localhost:8080/
   Key Concept: ReferencePolicy is in backend namespace, allows demo-apps access

6. 05-traffic-management.yaml (5 KB) - TRAFFIC MANAGEMENT
   Purpose: Learn weighted routing, canary, and blue-green deployments
   Contents:
   ├─ http-app-v2 Deployment (for testing)
   ├─ http-app-v2 Service
   ├─ Weighted route (70% v1, 30% v2)
   ├─ Canary route (95% v1, 5% v2)
   ├─ User-based canary (header-based)
   └─ Blue-green route (100% v1, ready to switch)
   
   Use: kubectl apply -f 05-traffic-management.yaml -n demo-apps
   When: After understanding routing basics
   Testing: Multiple requests to see traffic distribution
   Advanced: Patch weights to perform blue-green switch

setup.sh (2 KB) - AUTOMATED ONE-COMMAND SETUP
   Purpose: Automate kind cluster creation and Gateway API installation
   
   Steps:
   1. Delete existing kind cluster
   2. Create new kind cluster with config.yaml
   3. Install Gateway API CRDs
   4. Install Envoy Gateway helm chart
   5. Wait for controller readiness
   6. Verify installation
   
   Use: cd tutorials/gateway-api && ./setup.sh
   Time: ~2-3 minutes
   Requirements: Docker running, helm installed, kind installed
