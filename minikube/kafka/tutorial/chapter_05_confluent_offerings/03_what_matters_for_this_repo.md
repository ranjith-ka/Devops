# 5.3 What matters for this repo

References:

- https://developer.confluent.io/courses/apache-kafka/events/
- https://docs.confluent.io/platform/current/overview.html

For this repository, the most useful takeaway from the final Kafka 101 module is architectural, not branding-related.

## The practical lessons

- Kafka is the event backbone.
- Schema management becomes important once multiple teams share topics.
- Kafka Connect reduces custom integration code.
- Stream processing turns topics into continuously computed outputs.
- Operational tooling matters once the cluster is no longer a toy environment.

## Recommended next repo additions

- Schema Registry deployment
- Kafka Connect deployment
- example connectors
- sample event schemas
- stream-processing examples
